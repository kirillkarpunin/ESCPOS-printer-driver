# C++20 ESC/POS Lib

Библиотека для работы с чековыми термопринтерами по протоколу ESC/POS, написанная на C++20.

## Особенности

- Печать текста (с возможностью настройки размера, шрифта, стилей, отступов и проч.)
- Печать изображений

## Использование
```cpp
#include "ESCPOS/connection/ComPortConnection.h"
#include "ESCPOS/printer/Printer.h"

int main() {
    ComPortConnection c{"/dev/usb/lp0"}; // Создание проводного подключения
    Printer pr{c};

    pr.initialize(); // Инициализация принтера
    pr.print("Hello, world!");
}
```
```cpp
#include "ESCPOS/connection/BluetoothConnection.h"
#include "ESCPOS/printer/Printer.h"

int main() {
    BluetoothConnection c{"00:1A:2B:3C:4D:5E"}; // Создание bluetooth-подключения
    Printer pr{c};

    pr.initialize(); // Инициализация принтера
    pr.print("Hello, world!");
}
```
```cpp
#include "ESCPOS/connection/ComPortConnection.h"
#include "ESCPOS/printer/Printer.h"
#include "ESCPOS/image/Image.h"

int main() {
    ComPortConnection c{"/dev/usb/lp0"};
    Printer pr{c};
    pr.initialize();

    // Изображение "test_image.png" будет трансформировано в изображение размером 100x100
    Image im{"test_image.png", 100, 100} // Может быть png, jpg или bmp
    pr.print_image(im); // Печать изображения
}
```

## Сборка
Сборка осуществляется с помощью CMake.
```bash
# Клонирование репо
git clone git@github.com:kirillkarpunin/ESCPOS-printer-driver.git
cd ESCPOS-printer-driver

# Сборка
mkdir build && cd build
cmake ..
cmake --build .

# Сборка с тестами
mkdir build && cd build
cmake .. -DBUILD_TESTING=ON
cmake --build .
```

### Зависимости
- GCC 13+ / Clang 15+ (C++20 и \<format\>)
- CMake 3.31+
- libbluetooth
- nothings/stb (подтягивается автоматически с помощью CPM)
- GoogleTest (подтягивается автоматически с помощью CPM; только при сборке с тестами)

## Описание классов

#### `Connection`
Класс, описывающий подключение к принтеру. Реализует метод записи в файловый дескриптор, используемый для связи с принетром. Имеет двух наследников - `ComPortConnection` и `BluetoothConnection`. 

#### `ComPortConnection`
Класс, представляющий собой проводное подключение к принтеру. Имеет конструктор, принимающий строку - путь к файлу принтера в системе.

#### `BluetoothConnection`
Класс, представляющий собой подключение к принтеру через Bluetooth. Имеет конструктор, принимающий строку - MAC-адрес принтера.

---

#### `Printer`
Главный класс библиотеки, предоставляющий интерфейс для управления термопринтером. Имеет конструктор, принимающий объекты наследников класса `Connection`.

**Методы:**
- `void initialize() const` - Метод инициализации принтера, который **_необходимо вызвать перед работой с принтером_**. Очищает буфер печати и отменяет текущее форматирование.
<br>

- `void write(const std::string& data) const`
- `void write(const unsigned char& ch) const` - Добавляют строку или одиночный символ в буфер печати принтера без команды переноса строки.
- `void print() const` - Отправляет команду переноса строки, заставляя принтер напечатать все символы, находящиеся в буфере печати.
- `void print(const std::string& data) const`
- `void print(const unsigned char& ch) const` - Добавляют строку или одиночный символ в буфер печати принтера и отправляет команду переноса строки для немедленной печати.
<br>

- `void print_image(Image& image) const` - Печатает переданное изображение. **_В некачественных ESC/POS принтерах встречается баг, при котором печать изображения, высота которого кратна 96, вызывает загрязнение буфера печати принтера._**
<br>

- `void set_text_feature(Text::TextFeature feature, bool v)` - Включает (`true`) или выключает (`false`) стиль текста (полужирный, курсив, инверсия, подчеркивание).
- `void reset_text_features()` - Выключает все включенные стили текста.
- `bool get_text_feature(Text::TextFeature feature) const` - Возвращает текущее состояние конкретного стиля текста.
- `void set_text_underline_thickness(Text::UnderlineThickness v)` - Задает толщину линии подчеркивания - стандартная (`x1`) или двойная (`x2`).
- `Text::UnderlineThickness get_text_underline_thickness() const` - Возвращает текущую толщину линии подчеркивания.
- `void set_font(Text::Font v)` - Задает шрифт текста - стандартный (`A`) или альтернативный (`B`).
<br>

- `void set_text_width_scale(Text::Scale v)`
- `void set_text_height_scale(Text::Scale v)`
- `void set_text_scale(Text::Scale width, Text::Scale height)` - Задают множитель размера символов текста по ширине и/или высоте соответственно (от `x1` до `x4`).
- `void reset_text_scale()` - Сбрасывает множители размера символов текста по ширине и высоте к стандартному (`x1`).
- `Text::Scale get_text_width_scale() const`
- `Text::Scale get_text_height_scale() const` - Возвращают текущие множители размера символов текста по ширине и высоте соответственно.
<br>

- `void set_rotation(Text::Angle v)` - Устанавливает поворот печатаемого текста на `0`, `90`, `180` или `270` градусов.
- `void reset_rotation()` - Сбрасывает установленное значение поворота печатаемого текста к стандартному (`0` градусов).
<br>

- `void justify(Justification v)` - Устанавливает позицию печати текста: по левому краю, по центру или по правому краю.
- `void set_absolute_print_position(uint16_t v)` - Задает позицию начала печати текста (в точках) от левого края чековой ленты.
- `void set_right_side_character_spacing(uint8_t v)` - Задает промежуток (в точках) справа от каждого символа печатаемого текста.
- `void set_line_spacing(uint8_t v)` - Задает высоту межстрочного интервала (в точках).
- `void reset_line_spacing()` - Сбрасывает значение высоты межстрочного интервала к стандартному.
- `void set_left_margin(uint16_t v)` - Задает отступ левого поля (в точках) для всей области печати.
- `void set_print_area_width(uint16_t v)` - Задает ограничение ширины области печати (в точках).

#### `Text`
Структура для хранения текущих настроек печатаемого текста. Экземпляр этой структуры содержится в классе `Printer`.

Описывает ряд `enum`'ов, ожидаемых в методах класса `Printer`:
- `TextFeature` - Имеет значения `BOLD`, `ITALIC`, `INVERTED`, `UNDERLINED`.
- `Scale` - Имеет значения `x1`, `x2`, `x3`, `x4`.
- `Font` - Имеет значения `A`, `B`.
- `Angle` - Имеет значения `o0`, `o90`, `o180`, `o270`.
- `UnderlineThickness` - Имеет значения `x1`, `x2`.

#### `Justification`
`Enum`, содержащийся в классе `Printer`. Представляет собой набор возможных значений позиции печати текста. Имеет значения `LEFT`, `CENTER` и `RIGHT`.

---

#### `Image`
Класс, представляющий изображение для печати на принтере. Содержит логику загрузки файла изображения, изменения размера изображения и преобразования изображения в монохромный 1-битный формат с помощью алгоритма дизеринга. Для дизеринга используется матрица Байера размером 4x4.

Поддерживаются `png`, `jpg` и `bmp` изображения.

Конструктор класса ожидает:
- `const std::string& path` - Строка - путь к файлу изображения в системе.
- `int target_width = 0` - Целевая ширина изображения. При значении `0` (значение по умолчанию) целевой шириной считается исходная ширина изображения.
- `int target_height = 0` - Целевая высота изображения. При значении `0` (значение по умолчанию) целевой высотой считается исходная высота изображения.
- `double gamma = 1.0` - Параметр яркости изображения. При значениях `>1.0` изображение становится светлее; при значениях `<1.0` - темнее.

**Методы:**
- `void set_width_scale(Scale v)`
- `void set_height_scale(Scale v)`
- `void set_scale(Scale width, Scale height)` - Задают аппаратный множитель размера изображения по ширине и/или высоте соответственно (от `x1` до `x2`).
- `void reset_scale()` - Сбрасывает аппаратные множители размера изображения по ширине и высоте к стандартному (`x1`).
- `Scale get_width_scale() const`
- `Scale get_height_scale() const` - Возвращают текущие аппаратные множители размера изображения по ширине и высоте соответственно.

#### `Scale`
`Enum`, содержащийся в классе `Image`. Представляет собой набор возможных значений множителя размера изображения. Имеет значения `x1` и `x2`.

## Тестирование
**_Созданные тесты - это не совсем unit-тесты, поскольку они требуют физический принтер для запуска._**
При разработке тесты проводились на принтере 3Cott 3C-TP-58U.

Тестирование организовано с помощью библиотеки GoogleTest.

Для сборки с тестами необходимо использовать параметр `BUILD_TESTING=ON`.
```bash
# Сборка с тестами
mkdir build && cd build
cmake .. -DBUILD_TESTING=ON
cmake --build .
```

Путь к файлу принтера в системе или MAC-адрес принтера при запуске тестов указывается с помощью переменных окружения - `PRINTER_DEVICE_PATH` и `PRINTER_BT_ADDRESS` соответственно.
```bash
PRINTER_DEVICE_PATH="/dev/usb/lp0" PRINTER_BT_ADDRESS="00:1A:2B:3C:4D:5E" ./build/ESCPOS-testing
```

Тесты проводятся для обоих способов подключения. В случае, когда для какого-либо из способов не указана соответствующая переменная окружения, набор тестов для данного способа пропускается.
